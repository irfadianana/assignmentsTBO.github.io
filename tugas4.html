Regular Expression

Regular Expression yang biasanya disingkat menjadi Regex, merupakan sebuah teks (string) yang mendefinisikan sebuah pola pencarian sehingga dapat membantu kita untuk melakukan matching, locating, dan manipulasi teks.
Contoh Regular Expression :
> (0+1)* : himpunan seluruh string yang dapat dibentuk dari simbol '0' dan '1'
> (0+1)*00(0+1)* : himpunan string biner yang mengandung paling sedikit satu substring '00'
> (0+1)*00 : himpunan string biner yang diakhiri dengan '00'

Pemanfaatan Regular Expression

1. Bahasa Terkait dengan Regular Expression

Regular Expression dapat digunakan untuk mendeskripsikan beberapa bahasa sederhana. Jika r adalah Regular Expression, kita akan membiarkan L (r) menunjukkan bahasa yang terkait dengan r.
Bahasa L(r) yang dilambangkan dengan Regular Expression apapun, r ditentukan oleh aturan berikut :
1. ∅ adalah ekspresi reguler yang menunjukkan himpunan kosong,
2. λ adalah ekspresi reguler yang menunjukkan {λ},
3. Untuk setiap a ∈ Σ, a adalah ekspresi reguler yang menunjukkan {a}.
Jika r1 dan r2 adalah Regular Expression, maka
4. L (r1 + r2) = L (r1) ∪ L (r2),
5. L (r1 · r2) = L (r1) L (r2),
6. L ((r1)) = L (r1),
7. L (r ∗ 1) = (L (r1)) ∗.

2. Regex untuk Validasi Data

Regex dapat digunakan untuk matching atau pencocokan teks, contohnya seperti pada pengaturan password.
> Pertama-tama, kita harus mendefinisikan pola regex untuk data yang valid. Kemudian, kita lakukan pencocokan dengan pola tersebut.
> Contohnya seperti pada HTML dan Javascript yang ada pada code disamping
> Atribut pattern digunakan untuk menyimpan pola regex. Pada contoh di samping, polanya adalah pengecekan karakter yang diinputkan pada password.
> Pada contoh ini, regex tidak digunakan dalam code Javascript melainkan digunakan pada HTML.
> Code Javascript di samping, hanya bertugas untuk mengecek apakah nilai dari field password sudah sesuai atau tidak dengan yang ada pada atribut pattern

3. Regex untuk Mendeskripsikan Pola Sederhana

Setelah dieksplorasi, hubungan antara finite akseptor dan beberapa konstituen bahasa pemrograman yang lebih sederhana, seperti pengidentifikasi, atau bilangan bulat dan real. Hubungan antara finite automata dan regex berarti kita juga dapat menggunakan regex sebagai cara untuk mendeskripsikan fitur-fitur ini. Misalnya, dalam bahasa pemrograman, himpunan konstanta bilangan bulat ditentukan oleh regex sdd*, di mana s adalah tanda, dengan kemungkinan nilai dari {+, -, λ}, dan d adalah singkatan dari angka 0 sampai 9. Konstanta bilangan bulat adalah kasus sederhana dari apa yang biasanya disebut "pola", istilah yang merujuk pada sekumpulan objek yang memiliki beberapa properti umum. Pencocokan pola mengacu pada menetapkan objek tertentu ke salah satu dari beberapa kategori. Seringkali, kunci sukses pencocokan pola adalah menemukan cara yang efektif untuk mendeskripsikan pola.

Contoh:
Penerapan pencocokan pola terjadi dalam pengeditan teks. Semua editor teks memungkinkan file dipindai untuk mengetahui keberadaan string tertentu; kebanyakan editor memperluas ini untuk mengizinkan pencarian pola. Misalnya, editor vi dalam sistem operasi UNIX mengenali perintah / aba * c / sebagai instruksi untuk mencari file untuk kemunculan pertama dari string ab, diikuti dengan bilangan sembarang a, diikuti oleh c. Kami melihat dari contoh ini perlunya editor pencocokan pola untuk bekerja dengan ekspresi reguler.
Tugas yang menantang dalam aplikasi semacam itu adalah menulis program yang efisien untuk mengenali pola string. Mencari file untuk kejadian string tertentu adalah latihan pemrograman yang sangat sederhana, tetapi di sini situasinya lebih rumit. Kita harus berurusan dengan pola rumit sewenang-wenang yang jumlahnya tak terbatas; selanjutnya, pola tidak diperbaiki sebelumnya, tetapi dibuat pada waktu proses. Deskripsi pola merupakan bagian dari masukan, sehingga proses pengenalan harus fleksibel. Untuk mengatasi masalah ini, ide dari teori automata sering digunakan.
